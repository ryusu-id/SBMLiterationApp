## Configuration

Add the following section to your `appsettings.json` (fill in real values locally, never commit real credentials):

```json
"Email": {
  "SmtpHost": "smtp.gmail.com",
  "SmtpPort": 587,
  "SenderEmail": "your-sender-email@gmail.com",
  "SenderName": "SBM Literation App",
  "AppPassword": "your-gmail-app-password",
  "TestRecipientEmail": "your-test-recipient@gmail.com",
  "TestRecipientName": "Your Name (Test)"
}
```

### How to get a Gmail App Password

1. Go to [Google Account → Security](https://myaccount.google.com/security)
2. Enable **2-Step Verification** (required)
3. Go to **App passwords** → create one for "Mail"
4. Copy the 16-character password (with spaces) into `AppPassword`

> **Never commit real credentials.** Use `appsettings.Development.json` locally (it is gitignored) or environment variables / secrets manager in production.

## Endpoints

Both endpoints are **anonymous** (no auth required) for PoC testing purposes.

### 1. `POST /api/email/send-test`

Verifies SMTP connectivity with a simple test email.

**Request body** (both optional – falls back to appsettings):
```json
{
  "recipientEmail": null,
  "recipientName": null
}
```

Or send an empty body `{}` – it auto-uses `Email:TestRecipientEmail` from config.

**Success response:**
```json
{
  "message": "Test email sent successfully.",
  "data": {
    "message": "Test email sent successfully.",
    "sentTo": "your-test-recipient@gmail.com"
  }
}
```

---

### 2. `POST /api/email/group-assignment-preview`

Simulates a group-assignment notification email with **fully hardcoded mock data**.  
No request body needed — just hit the endpoint.

**Request:** No body required.

**What it does:**
- Constructs a mock assignment (`Week 3 Case Study – Supply Chain Strategy`)
- Sends a **personalised email to each participant** (2 accounts, both developer-owned for PoC)
- Each email has the same content but a different greeting (Hi **{FullName}**)

**Success response:**
```json
{
  "message": "[PoC] Mock group assignment notification sent successfully.",
  "data": {
    "message": "[PoC] Mock group assignment notification sent successfully.",
    "groupName": "Alpha Group",
    "participants": [
      "Your Name <your-test-recipient@gmail.com>",
      "Your Sender Account <your-sender-email@gmail.com>"
    ]
  }
}
```

**Error response (e.g. SMTP failure):**
```json
{
  "errorCode": "EMAIL_SEND_FAILED",
  "errorDescription": "<exception message>"
}
```

## PoC vs Production

| Concern | PoC (current) | Production (future) |
|---|---|---|
| Recipients | Hardcoded developer emails | Real group members from DB |
| Auth on endpoints | `AllowAnonymous` | `Roles("admin")` |
| Participant source | Hardcoded in endpoint | Queried from `GroupMember` + `User` |
| Email provider | Gmail SMTP | Gmail SMTP or SendGrid / Azure Communication Services |
| Credentials | appsettings (local only) | Environment variables / Azure Key Vault |
